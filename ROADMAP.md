# OpenGlaze Roadmap

## Near-term
- Community recipe contributions (YAML import workflow)
- Glazy / INSIGHT recipe import
- Frontend build pipeline (bundling, tree-shaking)
- Connection pooling and query optimization
- Blueprint-based route architecture

## Planned
- Mobile-responsive redesign
- Advanced glaze defect diagnosis (crazing, shivering, crawling, pinholing)
- Firing schedule builder with ramp/soak visualization
- Batch reporting with cost estimation
- Internationalization (i18n)

## Long-term
- Community recipe sharing (public recipe database)
- Studio collaboration features (role-based access, shared libraries)
- Advanced chemistry analysis tools (phase diagrams, eutectic prediction)
- Plugin system for custom materials and calculations

*This roadmap is a living document. Contributions and suggestions welcome via [GitHub Discussions](https://github.com/Pastorsimon1798/openglaze/discussions).*
